# Enhance Ability
Load the right context so the next task starts stronger than cold.
## What This Skill Does
Use this spell to apply a temporary, task-specific boost: priming a reviewer with the right files, warming caches before a demo, or loading domain context before a meeting. It improves the next move by front-loading exactly the capability the moment requires, not by pretending the system is globally smarter.
In this grimoire, Enhance Ability is treated as a metaphorical spell with a shipping-now delivery profile.
Canonical reference input: Enhance Ability (spell).
## When To Use

- A specific near-term task would benefit from context loading, warmup, or prep work.
- You want a bounded boost for the next move rather than a permanent change in capability.
- The bottleneck is cold start, missing context, or unprepared inputs rather than raw inability.

## Prerequisites

- No extra runtime dependencies beyond Hermes Agent and the normal toolset for this session.

## Procedure

1. Restate the target, the success condition, and any no-touch boundaries before taking action.
2. Identify the next task and the capability bottleneck holding it back.
3. Apply the smallest useful boost, such as context packets, cache warmup, examples, or briefing material.
4. Return what is boosted, how long it lasts, and what still needs normal effort.
5. Package the result as the deliverables below, with confidence, assumptions, and unresolved risk called out explicitly.

## Deliverables

- A bounded boost plan or prep bundle for the target task.
- A checklist of the contexts, resources, or conditions being enhanced.
- A statement of duration, limits, and what remains unchanged.

## Pitfalls / Guardrails

- Keep the metaphor anchored to a real mechanism instead of drifting into lore.
- Do not overstuff context or preload irrelevant material that degrades focus.
- Distinguish temporary readiness gains from actual model or system capability changes.

## Verification

- Check that the result includes every deliverable promised above.
- Check that confirmed facts, assumptions, and inferences are visibly separated.
- Check that the metaphor still maps cleanly to a real operational mechanism.

## Example Invocation
```text
/enhance-ability prime this system for the next task and tell me exactly what advantage the prep creates
```
