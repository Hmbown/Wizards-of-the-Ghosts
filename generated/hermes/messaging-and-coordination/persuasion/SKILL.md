# Persuasion
Craft clear, credible language that moves a person or team toward a decision.
## What This Skill Does
Use this skill when the problem is not information scarcity but alignment, framing, or buy-in.
In this grimoire, Persuasion is treated as a metaphorical skill with a shipping-now delivery profile.
Canonical reference input: Persuasion (skill).
## When To Use

- You need to write a convincing proposal, memo, comment, or pitch.
- A technically correct answer will fail without audience-aware framing.
- The goal is movement, not verbosity.

## Prerequisites

- No extra runtime dependencies beyond Hermes Agent and the normal toolset for this session.

## Procedure

1. Restate the target, the success condition, and any no-touch boundaries before taking action.
2. Identify the audience, incentives, objections, and decision threshold.
3. Build the argument around concrete benefits, tradeoffs, and credibility.
4. Use the tone that best matches the target person or forum.
5. Return a persuasive draft plus the core reasoning it relies on.
6. Package the result as the deliverables below, with confidence, assumptions, and unresolved risk called out explicitly.

## Deliverables

- A decision-oriented draft.
- A short objection-handling section.
- A note on which claims need evidence before sending.

## Pitfalls / Guardrails

- Keep the metaphor anchored to a real mechanism instead of drifting into lore.
- Do not manipulate through deception or fabricated certainty.
- Be explicit when the case depends on assumptions or incomplete evidence.

## Verification

- Check that the result includes every deliverable promised above.
- Check that confirmed facts, assumptions, and inferences are visibly separated.
- Check that the metaphor still maps cleanly to a real operational mechanism.

## Example Invocation
```text
/persuasion draft the strongest credible case for this proposal for the intended audience
```
