# Stealth

Move through a system with low visibility, minimal disruption, and a small footprint.

## Overview

Stealth is interpreted here as a metaphorical skill with a shipping-now execution model.

Canonical source: Stealth (skill)

Provider target: OpenClaw

## When To Use

- You want a read-only or low-noise pass before acting.
- A workflow needs discretion, minimal churn, or low observability.
- You need the smallest social or technical footprint possible.

## Workflow

1. Prefer passive observation before active modification.
2. Use the least noisy tools and the smallest relevant scope.
3. Avoid unnecessary notifications, edits, or visible side effects.
4. Report what was observed and what remained intentionally untouched.

## Deliverables

- A low-noise context report.
- A list of avoided side effects.
- A recommendation for when stealth is no longer appropriate.

## Guardrails

- Do not use stealth framing to justify unethical concealment.
- Respect policy, auditability, and explicit approval boundaries.

## Default Invocation

Use $stealth to gather the context we need with the smallest safe footprint and no unnecessary noise.
