# Persuasion

Craft clear, credible language that moves a person or team toward a decision.

## Overview

Persuasion is interpreted here as a metaphorical skill with a shipping-now execution model.

Canonical source: Persuasion (skill)

Provider target: OpenClaw

## When To Use

- You need to write a convincing proposal, memo, comment, or pitch.
- A technically correct answer will fail without audience-aware framing.
- The goal is movement, not verbosity.

## Workflow

1. Identify the audience, incentives, objections, and decision threshold.
2. Build the argument around concrete benefits, tradeoffs, and credibility.
3. Use the tone that best matches the target person or forum.
4. Return a persuasive draft plus the core reasoning it relies on.

## Deliverables

- A decision-oriented draft.
- A short objection-handling section.
- A note on which claims need evidence before sending.

## Guardrails

- Do not manipulate through deception or fabricated certainty.
- Be explicit when the case depends on assumptions or incomplete evidence.

## Default Invocation

Use $persuasion to draft the strongest credible case for this proposal for the intended audience.
