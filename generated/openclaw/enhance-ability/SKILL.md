# Enhance Ability

Load the right context so the next task starts stronger than cold.

## Overview

Enhance Ability is interpreted here as a metaphorical spell with a shipping-now execution model.

Canonical source: Enhance Ability (spell)

Provider target: OpenClaw

## When To Use

- A specific near-term task would benefit from context loading, warmup, or prep work.
- You want a bounded boost for the next move rather than a permanent change in capability.
- The bottleneck is cold start, missing context, or unprepared inputs rather than raw inability.

## Workflow

1. Identify the next task and the capability bottleneck holding it back.
2. Apply the smallest useful boost, such as context packets, cache warmup, examples, or briefing material.
3. Return what is boosted, how long it lasts, and what still needs normal effort.

## Deliverables

- A bounded boost plan or prep bundle for the target task.
- A checklist of the contexts, resources, or conditions being enhanced.
- A statement of duration, limits, and what remains unchanged.

## Guardrails

- Do not overstuff context or preload irrelevant material that degrades focus.
- Distinguish temporary readiness gains from actual model or system capability changes.

## Default Invocation

Use $enhance-ability to prime this system for the next task and tell me exactly what advantage the prep creates.
